# mrchafather — 프로젝트 지침

이 저장소는 **모바일 부고장을 정적 페이지로 만들어 GitHub Pages에 배포**하는 프로젝트입니다.
아래 지침은 세션과 무관하게 항상 준수합니다.

## 프로젝트 성격

- 원본은 상용 모바일 부고 서비스(`wooribugo4.com`)의 **동적 PHP 페이지**였음.
- **원본의 디자인·레이아웃을 그대로 복제**한다(원본 HTML + CSS/JS/폰트/이미지 자산을 내려받아
  `resources/` 하위에 그대로 두고, 절대경로 `/resources/`·`/upload/`를 상대경로로 변환).
- 동적 요소(PHP AJAX: 조문/근조화환/부의금 계좌 로드 등)는 정적 배포에서 동작하지 않으므로
  **호출을 비활성화**하고 안내 문구로 대체한다. 카카오 API 키 등 도메인 종속 자산은 재사용해도 무방하나
  기능 동작은 보장하지 않는다.
- 배포 대상: `https://dev-kjh.github.io/mrchafather/`

## PII 마스킹 정책 (필수 준수)

개인정보(PII)는 아래 규칙대로 처리한다. 원본을 다시 반영하거나 페이지를 재생성할 때도 **항상 적용**.

| 항목 | 처리 |
|---|---|
| 유족(상주) 휴대폰 번호 | **제거** — `tel:` 링크·번호 텍스트 모두 삭제. 연락 아이콘은 남기되 "연락처는 문자·카톡으로 안내" 문구/안내로 대체 |
| 부의금 계좌 | **미노출** — 계좌번호·예금주 등 데이터 표시 금지, 안내 문구만 |
| 장례식장 대표번호 | **유지** — 예: `031-772-1024` (`tel:` 링크 포함) |
| 고인·유족 이름 | **유지** (작성자 본인 가족 정보) |

> 원칙: 개인의 휴대폰 번호·금융 계좌 등 민감정보는 공개 페이지에 노출하지 않는다.
> 기관/시설의 대표 연락처는 유지한다.

## 파일 구조

- `index.html` — 원본을 복제한 부고장 본문 (경로 상대화 + PII 마스킹 + 동적 호출 비활성화)
- `resources/` — 원본에서 내려받은 CSS/JS/폰트/이미지 자산 (원본 디렉터리 구조 유지)
- `img/map.jpg` — 장례식장 위치 지도 이미지
- `README.md` — 페이지 설명 및 GitHub Pages 활성화 안내

> 원본 스타일을 바꾸지 말 것. 내용/일정 수정 시에도 원본 마크업·클래스 구조를 유지한다.

### CSS 경로 재작성 주의 (실수 방지)
원본 CSS의 `url()`은 **따옴표 있는 것/없는 것이 섞여** 있다. 절대경로 `/resources/`를 상대경로로 바꿀 때는
**경로 문자열만 치환**하고 따옴표는 건드리지 말 것.
- 올바름: `sed 's#/resources/#../../#g'` (따옴표 유무 그대로 유지)
- 잘못됨: `url(` 뒤에 따옴표를 임의로 삽입 → 따옴표 없던 `url(/resources/..)`가 `url("../..)`(닫는 따옴표 없음)이 되어
  **미종결 문자열이 그 지점 이후 CSS 전체를 무효화**시킴(상단바·레이아웃·배경 붕괴).
- 검증: `grep -E 'url\("[^"]*\)'` 와 `grep -E "url\('[^']*\)"` 결과가 **0**이어야 함.

## 자가호스팅 원칙 (장기 보관 목적 — 필수 준수)

이 페이지는 **오래 보관**하는 것이 목적이다. 외부 CDN·서비스는 언젠가 사라지거나 URL이 바뀔 수 있으므로,
**렌더링에 필요한 모든 정적 자산은 저장소 안(`resources/`)에 자가호스팅**한다.

**규칙**
1. **외부 URL로 리소스를 받지 않는다.** CSS/JS/HTML 어디에서도 `http(s)://`·`//cdn...`로 폰트·이미지·CSS·JS를
   불러오지 말 것. 전부 내려받아 `resources/` 하위에 두고 **상대경로**로 참조한다.
2. **폰트는 `@font-face`가 참조하는 모든 포맷(woff2/woff/ttf/otf/eot/svg)을 로컬에 갖춘다.**
   브라우저는 보통 woff2만 받지만, 레거시/포맷 우선순위에 따라 다른 포맷을 요청할 수 있어 404 방지를 위해 전부 보관.
3. **외부 SDK(카카오/밴드 등)는 자가호스팅하지 말고 제거**한다. 도메인·앱키에 종속되어 타 도메인에서 동작하지 않고
   수명도 불확실하므로, 관련 기능은 `alert('지원하지 않는 기능입니다.')` + 무해한 로컬 스텁(`window.Kakao=…`)으로 대체.
4. **예외 — 외부 "이동 링크"는 허용**: `map.kakao.com`(길찾기), `band.us/plugin/share`(공유)처럼
   리소스를 받아오는 게 아니라 사용자를 외부 서비스로 보내는 링크(`window.open`)는 자가호스팅 대상이 아니다.
5. **서버 API 호출 금지**: `.php`/`$.ajax` 등 원본 서버 동적 호출은 전부 제거하고, 필요한 데이터는
   `data/*.json` 등 **로컬 파일**로 스냅샷해서 클라이언트에서 렌더링(예: 조문/근조화환 목록·페이징).

**자산 추가/수정 시 검증 절차**
- 외부 참조 0 확인: `grep -rnoE "url\(['\"]?https?:|(src|href)=\"(https?:)?//" resources/ index.html`
  (결과에서 `map.kakao`/`band.us` 이동 링크만 남아야 함)
- 모든 CSS `url()`이 실제 로컬 파일로 연결되는지 스크립트로 전수 확인(누락 0).
- 헤드리스 렌더 후 **라이브(https)에서 404** 확인. GitHub Pages(리눅스)는 **대소문자 구분** →
  파일명 대소문자를 CSS 참조와 정확히 일치시킬 것(Windows 로컬은 통과해도 라이브에서 404 날 수 있음).
- 다른 페이지 전용 이미지(홈·앱스토어 배지·쿠폰 등)는 이 페이지 마크업에 요소가 없어 요청되지 않으므로 받지 않아도 됨.

## 커밋/푸시 전 민감정보 점검 (필수)

**모든 커밋·푸시 전에 민감정보 포함 여부를 반드시 점검한다.** 소스파일뿐 아니라 `CLAUDE.md`·`README.md`·
`data/*.json` 등 문서·데이터 파일도 포함해 확인한다.

**자동화(버전관리되는 git 훅)**
- `.githooks/pre-commit` 이 스테이징된 텍스트 파일을 스캔하고, 의심 항목이 있으면 **커밋을 차단**한다.
- 최초 1회 활성화 필요: `git config core.hooksPath .githooks`
- 점검 대상 패턴: 휴대폰 번호(010/011 등), 주민등록번호, 이메일, 32자리 앱키·인증 토큰류.
- 허용목록(오탐 방지): 장례식장 대표번호(031-772-1024), 지도 좌표. 새 예외는 훅의 `ALLOW`에 추가.
- 오탐이 확실할 때만 `git commit --no-verify` 로 우회(그 전에 반드시 육안 확인).

**수동 점검 원칙 (훅과 별개로 항상 수행)**
- 커밋 전 변경분을 확인: `git diff --cached` 로 새로 들어가는 값 눈으로 검토.
- 발견 시 처리: **유족 휴대폰·금융 계좌·주민번호·인증키 → 제거/마스킹**.
  (유족·조문객 실명, 장례식장 대표번호, 지도 좌표는 의도적으로 유지 — [[pii-policy]] 참고.)
- 외부에서 유입될 수 있는 민감정보(예: 원본 서버 계좌 조회 응답)는 애초에 **호출 코드를 두지 않는다**
  (서버 API 제거 원칙 참고). 한 번 원격에 올라간 값은 히스토리에 남으므로 **올리기 전 차단**이 핵심.

## 배포

- 정적 파일만 커밋/푸시.
- Pages 활성화: 저장소 **Settings → Pages → Source: `main` / `(root)`**.
  (`gh` CLI 미설치 환경에서는 브라우저에서 직접 설정)
